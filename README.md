# vite-plus-init

A modern Vue frontend starter based on **Vite+**, upgraded with:

- Vue 3 + TypeScript
- Vue Router
- UnoCSS
- unplugin-auto-import
- unplugin-vue-components

## Tech Stack

- Runtime UI: `vue`
- Routing: `vue-router`
- Utility CSS engine: `unocss`, `@unocss/vite`, `@unocss/preset-uno`, `@unocss/reset`
- DX plugins: `unplugin-auto-import`, `unplugin-vue-components`
- Toolchain: `vite-plus`

## Project Structure

```text
src/
	main.ts                # App entry (mount app + register router + UnoCSS)
	App.vue                # Root shell (RouterView)
	router/
		index.ts             # Router configuration
	pages/
		HomePage.vue         # Initial page
	components/            # Reusable components
	env.d.ts               # Vue + Vite type declarations
uno.config.ts            # UnoCSS config
vite.config.ts           # Vite+/plugin config
```

## Commands

Use Vite+ commands only:

- Install dependencies: `vp install`
- Start development: `vp dev`
- Check format/lint/types: `vp check`
- Build production: `vp build`
- Preview build output: `vp preview`

## Notes

- `src/auto-imports.d.ts` and `src/components.d.ts` are generated by unplugin after running `vp dev` or `vp build`.
- UnoCSS is enabled via `uno.config.ts` and imported in `src/main.ts`.

## Deploy To GitHub Pages

This project is configured with GitHub Actions for automatic static deployment.

1. Push code to the `main` branch.
2. Open repository Settings -> Pages.
3. Set Source to `GitHub Actions`.
4. Wait for workflow `Deploy To GitHub Pages` to complete.

Deployment workflow file:

- `.github/workflows/deploy-pages.yml`

Implementation details:

- `vite.config.ts` automatically sets `base` to `/<repo-name>/` in GitHub Actions.
- Router uses hash history to avoid 404 issues on page refresh in GitHub Pages.
